# 📘 Week 02: การตรวจสอบสถานะเครือข่ายและการทำงานของ Switch 
> *(Network Verification & Switch Operations)*

---

## 🎯 วัตถุประสงค์
1. **ฝึกฝนการใช้คำสั่งตรวจสอบสถานะ (Verification Commands):** เพื่ออ่านค่าคอนฟิกูเรชันปัจจุบันของอุปกรณ์เครือข่ายได้อย่างถูกต้อง
2. **เข้าใจความแตกต่างระหว่างสถาปัตยกรรม Layer 2 และ Layer 3:** แยกแยะบทบาทของ IP Address และ MAC Address ในระบบเครือข่ายได้
3. **เรียนรู้กลไกการทำงานเบื้องหลังของ Switch:** เข้าใจวิธีการส่งข้อมูลของ Switch ผ่านตารางความจำ (MAC Address Table)
4. **ฝึกทักษะการทดสอบและวิเคราะห์ปัญหาเชื่อมต่อเบื้องต้น:** สามารถระบุสาเหตุเมื่อเกิดความล้มเหลวในการสื่อสารภายในวง LAN

---

## 📖 เนื้อหาการเรียนรู้

### 1. เครื่องมือและคำสั่งสำหรับการตรวจสอบระบบ (Network Verification)
การเป็นวิศวกรเครือข่ายที่ดีไม่ได้เริ่มต้นที่การพิมพ์คอนฟิก แต่เริ่มต้นที่ **"การตรวจสอบสิ่งที่ตั้งค่าไว้ให้เป็น"** ผ่านคำสั่งสำคัญ 2 คำสั่งบน Cisco IOS:
* `show running-config` (หรือ `sh run`): คำสั่งเปิดดูสมุดบันทึกการตั้งค่าทั้งหมดที่อุปกรณ์กำลังใช้งานอยู่ในหน่วยความจำ (RAM) ณ ปัจจุบัน
  > ⚠️ **ข้อควรจำ:** หากผลลัพธ์ยาวเกินหน้าจอ จะปรากฏคำว่า `--More--` ด้านล่าง ให้กดปุ่ม **Spacebar** เพื่อดูหน้าถัดไป หรือกด **Ctrl + C** เพื่อยกเลิก
* `show ip interface brief` (หรือ `sh ip int br`): คำสั่งสรุปสถานะของพอร์ตเชื่อมต่อทั้งหมดแบบย่อ ช่วยให้เช็คได้ทันทีว่าพอร์ตไหนเปิด (`up`) หรือพอร์ตไหนปิด (`down/administratively down`)

### 2. อัตลักษณ์ของอุปกรณ์: IP Address vs MAC Address
ในการสื่อสารข้อมูล อุปกรณ์ทุกชนิดในเครือข่ายจะมี "สองอัตลักษณ์" เสมอ:
* **IP Address (Logical Address):** เปรียบเสมือน **"บ้านเลขที่"** สามารถเปลี่ยนแปลงได้ตามเครือข่ายที่ไปเชื่อมต่อ ทำหน้าที่ระบุว่าอุปกรณ์อยู่หมู่บ้านไหน (Network ID) และเครื่องไหน (Host ID)
* **MAC Address (Physical Address):** เปรียบเสมือน **"เลขประจำตัวประชาชน"** เป็นค่าฮาร์ดแวร์ที่ถูกฝังมาในการ์ดแลน (NIC) จากโรงงาน มีค่าเดียวในโลกและไม่สามารถเปลี่ยนแปลงได้

### 3. Switch ทำงานอย่างไร? (Inside the Switch)
Switch เป็นอุปกรณ์ใน Layer 2 มันไม่สนใจ IP Address แต่คุยกันด้วย MAC Address 
* เมื่อมีข้อมูลส่งเข้ามาในพอร์ต Switch จะจดจำเลข MAC Address ของคอมพิวเตอร์ต้นทางคู่กับหมายเลขพอร์ตนั้นลงในสมุดบันทึกที่เรียกว่า **"MAC Address Table"**
* เมื่อส่งข้อมูลครั้งต่อไป Switch จะเปิดตารางนี้ดู ทำให้สามารถส่งข้อมูลตรงไปยังพอร์ตปลายทางได้อย่างแม่นยำ ไม่ส่งกระจายไปกวนเครื่องอื่น
* เราสามารถเปิดดูตารางนี้ได้ด้วยคำสั่ง `show mac address-table`

**ไฟล์ปฏิบัติการ:** `week2_test_config.pkt` *(ประกอบด้วย Switch 1 ตัว และ PC 5 เครื่อง)*

### 📍 ขั้นตอนการทำแล็บ

#### สเต็ปที่ 1: ตรวจสอบฝั่งคอมพิวเตอร์ (End Devices)
1. คลิกที่คอมพิวเตอร์แต่ละเครื่อง (**PC0 - PC4**) -> ไปที่แท็บ **Desktop** -> เข้า **Command Prompt**
2. พิมพ์คำสั่ง `ipconfig /all` แล้วกด Enter
3. **บันทึกผล:** นำค่า IP Address, Subnet Mask และ MAC Address ของคอมพิวเตอร์ทุกเครื่องลงในใบงาน

#### สเต็ปที่ 2: ตรวจสอบพอร์ตเชื่อมต่อบน Switch
1. คลิกที่ตัว **Switch1** -> ไปที่แท็บ **CLI** -> กด Enter 1-2 ครั้ง
2. พิมพ์คำสั่ง `enable` เพื่อสิทธิ์ผู้บริหารระบบ (เครื่องหมายเปลี่ยนเป็น `Switch#`)
3. พิมพ์คำสั่ง `show ip interface brief`
4. **บันทึกผล:** ตรวจดูว่าพอร์ต FastEthernet 0/1 ถึง 0/5 มีสถานะในช่อง Status และ Protocol เป็นคำว่า `up` หรือไม่

#### สเต็ปที่ 3: ทดสอบทางเดินข้อมูล (Connectivity Test)
1. กลับไปที่ **PC0** -> เข้า **Command Prompt**
2. พิมพ์คำสั่ง `ping 192.168.1.2` เพื่อทดสอบสัญญาณไปยัง PC1
3. ทำการทดสอบ Ping ไล่ไปให้ครบทุกเครื่องในเครือข่าย
4. **บันทึกผล:** นำผลลัพธ์ไปกรอกลงในตาราง Ping Matrix 

#### สเต็ปที่ 4: เจาะลึกความจำของ Switch
1. กลับมาที่ **Switch1** ในแท็บ **CLI** (โหมด `Switch#`)
2. พิมพ์คำสั่ง `show mac address-table` แล้วกด Enter
3. **บันทึกผล:** สังเกตว่า Switch สามารถเรียนรู้ MAC Address ของ PC แต่ละเครื่องได้ตรงกับพอร์ตที่เสียบสายอยู่จริงหรือไม่

## 🛠️ คลังคำสั่งที่ใช้ตรวจสอบและแก้ไขปัญหา

```bash
# ========================================================================
# คำสั่งสำหรับใช้งานบน PC (ผ่าน Command Prompt)
# ========================================================================
ipconfig /all                 # ดูข้อมูล Network และ MAC Address ของตัวเอง
ping [IP Address]             # ทดสอบการส่งแพ็กเก็ตไปยังอุปกรณ์ปลายทาง

# ========================================================================
# คำสั่งสำหรับใช้งานบน Switch (ผ่าน CLI โหมด Privileged EXEC '#')
# ========================================================================
Switch# show running-config       # เปิดดูการตั้งค่าปัจจุบันทั้งหมดใน RAM
Switch# show ip interface brief   # เช็คสถานะพอร์ตเชื่อมต่อทั้งหมดแบบย่อ
Switch# show mac address-table    # ดูตารางความจำพอร์ตและ MAC Address
